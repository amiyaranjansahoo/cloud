# ROLES:
```sh
•	Roles are Similar to users, but they are intended for temporary use by applications, services or other entities.
•	If You want to access Roles we need one permission called STS( Security Token Service)
•	IAM roles provide a secure and flexible way to manage access to AWS resources without the need for permanent
credentials.
```
## Types of roles based on entity type
```sh
•	AWS Service—->you can call aws other services by using specific service.
•	AWS Accounts
•	Web Identity
•	SAML (Security Assertion Markup Language)
•	Custom Role.
```
